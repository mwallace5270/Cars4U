Cars4U 

Application Description: 
    A collection of scripts that make up a web app designed faciltate buying a car (cannot currently buy car; 
    we didn't get there). Buyers can browse, save and buy cars. Once the buyer finds a cart they would like to 
    purchase they will be directed to checkout to input their payment information.  
Core Features: 
    - User Registration and Login: Buyers create an account to view active car listings, add 
    cars to checkout and purchase the car. 
    - Profile/Data Deletion: Users can choose to delete thier accounts when no longer necessary/wanted. 
    - Car Listings - Buyers can browse the listings and see the specs of the car (make, model, 
    year, mileage), and view a picture of the car.  
    - Favorites: Users are able to favorite the cars they like and view them later. 
    
Unavailable Features:  
    - Filetering: Users are able to filter the cars displayed. 
    - Multiple Car Images: Buyers are able to view multiple images of the car(s) they're interested in. 
    - Seller Management - Buyers can see the dealerships information, reviews and all the active 
    cars they have listed.  
    - Order History and Profile - Buyers can see their past purchases and current orders. 
    - Payment Gateway - Buyers purchase the vehicle via a payment gateway.   

Known Bugs/Limitations: 
    - Currently the header is not diplaying the way I would like it to. 
    - The payment gateway does not store info and is just there as a form. 
    - Not sure if all the pages are linked together properly.
